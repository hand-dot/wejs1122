関数型プログラミングで重要な概念の1つである  
`Higher-Order Function`

---
JavaScriptやどの関数型言語においても  
関数は「値」なんだ  

---
```js
function double (num){ 
    return num * 2; 
} 
```
これはどんな言語でもできる

---
```js
const double = function(num){
    return num * 2;
}
```
これはすべての言語でできるわけじゃない  
<small>無名関数を作って変数に代入</small>

---
しかも、別の関数に渡すこともできる。  
これが`Higher-Order Function`

---
これを使えば関数を合成(Composition)できる

普通のforループを使って取得する例

```js
let underTen = [];
for(let i = 0; i < sazaeCharacters.length; i++){
    if(sazaeCharacters[i].age < 10){
        underTen.push(sazaeCharacters[i]);
    }
}
```
6行, 122文字

---
`Array.prototype.filter()`を使って取得する例
```js
const isUnderTen = user => user.age < 10;
const underTen = sazaeCharacters.filter(isUnderTen);
```
2行, 83文字  
見た目だけではなく、ロジックが減っている。  
<small>新しい配列を作って代入する処理を書いていない。</small>

---
すごく単純な例だけど、この処理には  

* 10歳未満かどうかを決定するという処理  
* 配列を作ってオブジェクトを入れていくという処理  

が含まれていて  
`isUnderTen`と`filter`という2つの単純な関数で別々に考えられた
<small>forループの例みたいに2つの問題がごちゃ混ぜになっているときより理解しやすいし、再利用しやすい。</small>
